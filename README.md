# **Проекты**

## *Уровень 1: Консольные приложения*

1. **Калькулятор с историей операций.**
   - **Этап 1:** Линейная структура для одной операции.
   - **Этап 2:** Ветвление для выбора операции (+, -, *, /).
   - **Этап 3:** Цикл для непрерывной работы до команды "выход".
   - **Этап 4:** Добавить функции для каждой операции.
   - **Этап 5:** Сохранение истории вычислений в файл (файлы последовательного доступа).

2. **Игра "Угадай число".**
   - **Этап 1:** Компьютер загадывает, пользователь угадывает с подсказками "больше/меньше".
   - **Этап 2:** Добавить подсчёт попыток и рейтинг игроков.
   - **Этап 3:** Сохранение таблицы рекордов (имя, количество попыток) в файл.

3. **Система управления библиотекой (мини-версия).**
   - Работа со структурой `Book` (автор, название, год, наличие).
   - Массив/вектор структур для хранения каталога.
   - Функции: добавление, поиск по автору/названию, вывод всего каталога, отметка о выдаче.
   - Сохранение и загрузка каталога из типизированного файла.

4. **Анализатор текстового файла.**
   - Чтение файла. Подсчёт: количество строк, слов, символов.
   - Поиск частоты использования заданного слова.
   - Вывод статистики (самое частое/длинное слово).

5. **Программа для работы с многочленами.**
   - Структура для хранения коэффициентов.
   - Функции: сложение, вычитание, умножение многочленов.
   - Рекурсивное вычисление значения по схеме Горнера.

## *Уровень 2: Углубление, алгоритмы и структуры данных*

1. **Архиватор (простейший по алгоритму RLE).**
   - Реализация алгоритма Run-Length Encoding.
   - Создание модуля для работы с алгоритмом (отдельные `.h` и `.cpp` файлы).
   - Подсчёт степени сжатия.

2. **Игра "Сапёр" (консольная версия).**
   - Двумерный массив для поля.
   - Алгоритмы расстановки мин и подсчёта чисел.
   - Рекурсивное открытие пустых областей.
   - Цикл игры с обработкой ввода координат.

3. **To-Do List (Список дел) с динамическими структурами.**
   - Реализация на основе односвязного списка (структура `Task` + указатель `next`).
   - Функции: добавление в начало/конец, удаление, отметка о выполнении, просмотр всех.
   - Усложнение: Двусвязный список, стек для отмены действий.

4. **Калькулятор для выражений в постфиксной записи (Обратная польская нотация).**
   - Реализация стека на основе массива или односвязного списка.
   - Алгоритм вычисления выражения вида `5 3 +` -> `8`.
   - Усложнение: Алгоритм перевода из инфиксной (`(5+3)*2`) в постфиксную запись.

## *Уровень 3: Объектно-ориентированное программирование и GUI*

1. **Графический калькулятор.**
   - Создание формы с кнопками и полем ввода.
   - Обработка событий нажатия кнопок.
   - Логика вычислений, инкапсулированная в отдельном классе `CalculatorEngine`.
   - Использование стандартных диалогов для вывода ошибок.

2. **Анкета или тестирующая система.**
   - Несколько форм: форма ввода данных, форма с вопросами (использовать `RadioButton`, `CheckBox`), форма с результатом.
   - Класс `Test`, содержащий список вопросов и ответов.
   - Навигация между формами.
   - Усложнение: Таймер на прохождение теста.

3. **Игра "Пятнашки" (15 Puzzle).**
   - Класс `GameBoard`, отвечающий за логику поля и перемещения костяшек.
   - Создание интерфейса с кнопками или рисование поля на канве.
   - Обработка кликов для перемещения.
   - Проверка на завершение игры. Меню ("Новая игра", "Выход").

4. **Клиент для записной книжки (сетевая версия - как долгосрочный проект).**
   - Фронтенд (GUI): Форма для просмотра, добавления, редактирования контактов (структура `Contact`).
   - Бэкенд (логика + файлы): Отдельный класс `NotebookService` для управления данными, работа с файлами.
   - ООП: Чёткое разделение на классы Model (`Contact`, `NotebookService`), View (форма), Controller (обработчики событий).
   - Тестирование: Написание unit-тестов (например, с помощью библиотеки Google Test) для методов `NotebookService`.

5. **Симулятор банкомата.**
   - Многомодульная структура: Модуль для работы с аккаунтами (`Account`), модуль для транзакций, модуль интерфейса.
   - GUI: Форма ввода PIN, главное меню, операции (баланс, снятие, перевод).
   - Работа с данными: Хранение данных о картах, счетах и транзакциях в бинарных файлах прямого доступа (для быстрого поиска по номеру карты).
   - Безопасность: Шифрование PIN-кодов (простейшее, для демонстрации).
   - Логирование: Запись всех операций в лог-файл.
   - Тестирование: Юнит-тесты для критических модулей.
